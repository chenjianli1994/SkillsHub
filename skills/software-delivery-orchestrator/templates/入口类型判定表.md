# 入口类型判定表

| 入口类型 | 典型触发语句 | 必要输入 | 后续动作 |
| --- | --- | --- | --- |
| `full-flow` | 从需求开始完整跑一遍 | 原始需求或需求材料 | 需求分析到质量门禁 |
| `full-flow-with-code` | 从需求一路跑到测试通过的代码，并据此回填详设/策略 | 原始需求或需求材料 | 需求分析…任务计划、代码实现、代码驱动回填详设/策略、质量门禁 |
| `requirement-only` | 只生成需求规格书 | 需求材料 | 需求分析、需求整理、质量门禁 |
| `codebase-only` | 只分析当前代码 | 代码仓库 | 代码分析 |
| `design-from-code` | 基于当前代码生成设计文档 | 代码仓库 | 代码分析、方案设计、详细设计 |
| `strategy-from-code` | 基于当前代码生成策略文档 | 代码仓库或代码片段 | 代码分析、策略文档 |
| `reference-doc-driven` | 参考这份文档生成 | 参考文档 | 参考文档画像、目标阶段 |
| `design-from-code-with-reference` | 参考文档格式并基于代码生成设计 | 代码仓库、参考文档 | 画像、代码分析、设计文档 |
| `strategy-from-reference` | 参考策略文档生成新策略 | 参考策略文档、新输入 | 画像、策略文档 |
| `requirement-from-reference` | 参考需求文档生成新需求 | 参考需求文档、新输入 | 画像、需求文档 |
| `export-with-reference-style` | 参考 Word 格式导出 | 源文档、参考文档、目标格式 | 画像、文档导出 |
| `task-plan-from-design` | 根据设计拆任务 | 设计文档 | 任务计划 |
| `implement-from-plan` | 按任务计划/详细设计生成代码并测试通过 | 任务计划或详细设计 | 代码实现、（可选）代码驱动回填文档 |
| `quality-only` | 检查已有文档质量 | 已有产物 | 质量门禁 |
| `export-only` | 导出 Word/PDF | 源文档、目标格式 | 文档导出 |

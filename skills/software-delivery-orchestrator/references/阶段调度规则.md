# 阶段调度规则

## 基本原则

- 总工作流只做编排，不替代阶段 skill。
- 阶段 skill 不存在时，记录阻塞项，不伪造产物。
- 用户可见产物写入 `outputs/`。
- 内部 JSON、状态文件和画像写入 `state/`。

## 缺失 skill 处理

| 缺失项 | 处理方式 |
| --- | --- |
| `codebase-analysis-reporter` | 记录“代码分析 skill 尚未建立”，停止代码驱动链路 |
| `solution-architecture-designer` | 记录“方案设计 skill 尚未建立”，不生成方案文档 |
| `detailed-design-writer` | 记录“详细设计 skill 尚未建立”，不生成详细设计 |
| `implementation-task-planner` | 记录“任务计划 skill 尚未建立”，不生成任务计划 |
| `software-quality-gate` | 使用已有质量检查 skill 可覆盖的部分，并记录缺口 |

## 输出边界

`outputs/` 只能放用户需要的产出物。以下内部产物不得写入 `outputs/`：

- 工作流状态 JSON
- 结构化阶段 JSON
- 参考文档画像 JSON
- `state/` 下的内部画像 Markdown
- 内部调度记录

如果用户明确要求查看参考文档画像，应生成面向评审的独立 Markdown 交付物，不要把 `state/` 内部文件直接复制到 `outputs/`。

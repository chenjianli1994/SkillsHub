# MCP 配置说明

## 用途

本目录用于保存项目级 MCP 配置示例和后续接入说明。当前项目不依赖全局配置，后续可以按需接入企业文档库、数据库、搜索服务、需求管理系统等外部知识来源。

## 文件说明

- `mcp-config.example.json`：示例配置，不包含真实密钥或生产地址。

## 接入原则

- 不要把真实账号、Token、数据库密码写入仓库。
- 真实凭据应通过环境变量、密钥管理服务或本机安全配置注入。
- MCP 返回的信息应作为补充来源，不能覆盖项目内 `knowledge/` 的明确规则。
- Agent 执行需求分析时，应先读取 `knowledge/` 和项目内 skills，再按需调用 MCP。

## 可扩展方向

- 企业文档库：读取制度、产品说明、接口文档、历史需求。
- 数据库：查询业务字典、枚举值、字段含义和数据样例。
- 搜索服务：跨文档检索业务资料和历史决策。
- 需求管理系统：同步需求条目、缺陷、版本计划和验收记录。

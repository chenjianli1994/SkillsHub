# 汽车软件需求检查表

## 输入与分类

- [ ] 原始材料来源可定位。
- [ ] 确定事实、合理推断和待确认假设分开。
- [ ] 未知参数没有被补写。

## 领域控制

- [ ] 温度阈值及单位有批准来源，否则为待确认。
- [ ] 时间、采样、滞回和恢复条件有批准来源，否则为待确认。
- [ ] 信号名、范围和更新条件有批准来源，否则不创建。
- [ ] 传感器异常覆盖判定、上报影响、降级动作和恢复条件。
- [ ] 故障等级和安全参数没有由模型推断。

## 需求质量

- [ ] FR/DR/PR/NFR 编号唯一。
- [ ] FR 有触发、前置、规则、输入、输出、异常和验收。
- [ ] 每条需求有来源、验证方式和追踪状态。
- [ ] 冲突来源、影响和裁决责任人被保留。

# 动力电池冷却控制领域知识（脱敏示例）

## 1. 适用范围

本文件仅用于演示需求分析助手的知识检索和缺口识别，不是某车型、某企业或某 ECU 的批准控制规范。文件不定义具体温度、时间、信号名、故障等级、降级参数或安全目标。

## 2. 对象与意图

- 动力电池冷却控制关注电池热状态、冷却能力、系统运行状态和异常状态之间的关系。
- 正常控制意图是根据已批准的电池热状态规则提出或调整冷却请求。
- “温度偏高”“增强冷却”等词描述控制意图，不等于已经给出可执行阈值。
- 需求应说明输入数据、状态判定、控制动作、输出状态和异常处理之间的追踪关系。

## 3. 传感器异常处理框架

传感器异常需求至少需要分别确认四类信息：

1. 异常判定：什么数据状态或通信状态被视为无效。
2. 上报与影响：异常如何通知上层，哪些功能受到影响。
3. 降级动作：异常时采用哪一种已批准的处置路径。
4. 恢复条件：何时允许恢复正常控制，是否需要确认或滞回。

如果上述任一项没有批准来源，需求输出只能保留待确认字段。不能根据通用经验填写替代值、故障等级、切换时间或恢复阈值。

## 4. 需求分析提示

- 看到温度阈值但没有边界或单位时，建立 P0 确认问题。
- 看到“及时”“快速”“安全”等词时，要求补充可验证口径。
- 看到两个来源给出不同控制动作时，保留冲突来源和影响，不自动选边。
- 只有说明控制目标而没有输入、状态或验收时，标记材料不完整。

## 5. 明确未提供的信息

本脱敏示例未提供：温度数值和单位、采样或滤波参数、信号名称与编码、冷却执行器接口、故障等级、替代值、降级动作参数、恢复条件、滞回和安全目标。上述信息必须由系统、软件、测试或功能安全责任方确认。
